## рҹ”Ҙ The Problem

Startups fail not from bad ideas вҖ” they fail from **bad resource allocation**.

Every week, managers across thousands of small teams face the same problem:

> *"Who should build this feature? Who has time? Who actually has the right skills?"*

They guess. They overload their best performers. They underutilize people with niche skills. Tasks fail вҖ” not because the team was incapable, but because the **wrong person was assigned**.

No existing tool solves this for small teams:

| Tool | What it does | What it misses |
|---|---|---|
| Jira | Tracks tasks | Can't match tasks to skills |
| Notion | Documents everything | No allocation intelligence |
| Asana | Manages projects | Rule-based, not semantic |
| Monday.com | Visualizes work | Designed for enterprise, not startups |

**HireRight fills this gap** вҖ” a lightweight, AI-native task allocation engine built specifically for 2вҖ“50 person teams.

---

## рҹ’Ў The Solution

HireRight takes your **team's skills** and your **task list**, and uses AI to match them вҖ” automatically, intelligently, and with full explainability.

```
You provide:                    HireRight delivers:
в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ                   в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
Team members + skills     вҶ’     Smart assignments
Tasks + requirements      вҶ’     Confidence scores  
Project deadlines         вҶ’     AI reasoning
Budget/hour estimates     вҶ’     Workload balance
                          вҶ’     Risk warnings
```

Every assignment comes with a **confidence score** and a **plain-English explanation** of why that person was chosen вҖ” so managers can trust, verify, or override the AI's decision.

## вңЁ Key Features

### рҹ”җ Authentication System
- Secure user registration and login
- JWT-based session management (7-day tokens)
- bcrypt password hashing
- Protected routes вҖ” your data is only yours

### рҹ“Ғ Multi-Project Workspace
- Create unlimited projects per account
- Each project has isolated team, tasks, and assignments
- Project-level demo data seeding
- One-click project reset for clean re-runs

### рҹ§  AI-Powered Matching
- **TF-IDF vectorization** via scikit-learn вҖ” lightweight and fast
- **Availability-aware** вҖ” never overloads a team member
- **Confidence scoring** вҖ” 0вҖ“100% match quality
- **Greedy optimization** вҖ” tasks assigned in priority order

### рҹ“Ҡ Assignment Dashboard
- Real-time workload distribution chart
- Color-coded confidence levels (green/yellow/red)
- Claude-generated team health summary
- Per-assignment AI reasoning

### вҡЎ One-Click Auto-Assign
- Runs full allocation in 5вҖ“15 seconds
- Handles unassigned tasks gracefully
- Fully resetable and re-runnable

## рҹҡҖ Getting Started

### Prerequisites

```
вң… Python 3.10 or higher
вң… Node.js 18 or higher
вң… An Anthropic API key вҶ’ console.anthropic.com
```

### 1. Clone the Repository

```bash
git clone https://github.com/RudraPyt/HireRight.git
cd HireRight
```

### 2. Backend Setup

```bash
cd backend

# Install dependencies
pip install -r requirements.txt

# Create environment file (Windows)
echo ANTHROPIC_API_KEY=your_key_here > .env
echo SECRET_KEY=your_random_secret_here >> .env

# Start the server
uvicorn main:app --reload --port 8000
```

вң… Backend running at: `http://localhost:8000`  
вң… API docs at: `http://localhost:8000/docs`

### 3. Frontend Setup

Open a second terminal:

```bash
cd frontend

# Install dependencies
npm install

# Start the dev server
npm run dev
```

вң… Frontend running at: `http://localhost:5173`

### 4. First Run

```
1. Open http://localhost:5173
2. Click "Create Account" вҶ’ register
3. Click "New Project" вҶ’ create your first project
4. Go to Team tab вҶ’ add members with skills
5. Go to Tasks tab вҶ’ add tasks with requirements
6. Click "Auto-Assign" вҶ’ watch the AI work
```

> рҹ’Ў **Quick Demo:** Click "Load Demo Data" to instantly populate 5 team members and 6 realistic tasks, then hit Auto-Assign.

## рҹ“Ў API Reference

Base URL (Production): `https://hireright-backend.onrender.com`  
Base URL (Local): `http://localhost:8000`  
Interactive Docs: `https://hireright-backend.onrender.com/docs`

### Authentication

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/auth/register` | Register new user |
| `POST` | `/auth/login` | Login, receive JWT token |
| `GET` | `/auth/me` | Get current user info |

### Projects

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/projects` | List all user's projects |
| `POST` | `/projects` | Create new project |
| `GET` | `/projects/{id}` | Get project details |
| `DELETE` | `/projects/{id}` | Delete project + all data |

### Team Members

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/projects/{id}/members` | List project members |
| `POST` | `/projects/{id}/members` | Add team member |
| `DELETE` | `/projects/{id}/members/{mid}` | Remove member |

### Tasks

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/projects/{id}/tasks` | List project tasks |
| `POST` | `/projects/{id}/tasks` | Add task |
| `DELETE` | `/projects/{id}/tasks/{tid}` | Remove task |

### AI Allocation

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/projects/{id}/assign` | Run AI matching |
| `GET` | `/projects/{id}/assignments` | View assignments |
| `GET` | `/projects/{id}/workload` | Workload per member |
| `POST` | `/projects/{id}/reset` | Clear all assignments |
| `GET` | `/projects/{id}/seed-demo` | Load demo data |

### Example Request

```bash
# Login
curl -X POST https://hireright-backend.onrender.com/auth/login \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=rudra&password=mypassword"

# Response
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer",
  "username": "rudra"
}

# Run Auto-Assign (with token)
curl -X POST https://hireright-backend.onrender.com/projects/1/assign \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```
